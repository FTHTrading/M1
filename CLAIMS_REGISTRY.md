#### CLM-014 — "Graceful shutdown and zero message loss"

| Field | Value |
|-------|-------|
| **Support** | SUPPORTED |
| **Confidence** | HIGH |
| **Source** | M1/README.md |

**Evidence**:
- `apps/worker/src/index.ts` — `SIGTERM`/`SIGINT` handlers call `worker.close()` on all
  BullMQ workers before process exit
- BullMQ's `close()` waits for in-flight jobs to complete
- `Promise.all([w1.close(), w2.close(), ...])` pattern ensures all workers drain

**Analyst note**: This is the correct implementation. Message loss requires a worker to die
mid-job; the graceful shutdown pattern prevents that in all normal termination scenarios.
